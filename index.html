What is a "cross platform" app?

  • Framework that allows you build iOS and Android apps with 1 code base
  • Some of them even state native performance!
  • Wait... but truly native performance?

Example: Think Electron but for mobile + web rather

Why Cross Platform?

Savings


  • With the ability to generate a web, Android, and iOS app from the same code base this one is obvious
  • The savings don't only come from a quicker project timeline
  • You also need fewer developers for MVP's
  • You could also use more developers and build more robust applications

Faster Developer Production


  • Nowadays everyone wants an app
  • Most developers aren't keen on learning 2 lanugages to build an app
  • They utilize web technologies so you can use your existing skillset

Consistent Look and Feel


  • Oten times we hear clients say "Can you make the Android button look like the iOS button?"
  • Most of the time your UI will be very similar across platforms (iOS and Android)

Performance


  • Yes, performance
  • Note: Native apps will ALWAYS be more performant than cross platform frameworks
  • Some frameworks get very close to native performance
  • Some are able to achieve it in most cases
  • For 1/2 - 1/3 of the cost of a native app they do get pretty impressive performance for the price

Maintenance


  • The iOS and Android apps can usually be maintained by 1 developer
  • Developers switch companies...
  • If you can use that web experience then it will be easier for them to maintain the app

Continue to use web resources


  • Since most of them use JavaScript you can continue to use JavaScript resources:
  • npm packages
  • web resources (api's)
  • You could use these in the native apps as well but the look and feel will be almost identical to the web app
  • There are some caveats to this! (See NativeScript)
